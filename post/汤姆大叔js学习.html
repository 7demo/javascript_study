变量：

全局变量是在任何函外面声明的或者是未声明可以直接简单使用的
this指该环境的全局对象。在浏览器中，全局对象有window属性，但是window指全局对象本身

减少全局变量：

实用var声明；
命名空间模式
自执行函数

若在js环境内不加var使用局部变量，则会隐式转成全局变量，特别注意的是实用任务链进行声明时：
function foo () {
var a = b = 0;
};
此时，由于是从右到左进行赋值的，也就是先b = 0，但由于b未声明，则隐式的转变为全局变量（隐式全局变量），最好如下：
function foo () {
var a,b;
a = b = 0
};

隐私全局变量与明确定义全局变量的区别：

通过var创建的全局变量是不能被delete删除的；
隐式全局变量是可以被delete删除的；
;
说明隐式全局变量并不是真正全局变量，它们是全局对象的属性。而属性是可以通过delete删除的，而变量是不能的。具体到执行上：

var global_var = 1;
global_novar = 2;

执行delete时，返回值也不同，删除真正全局变量返回false，隐式全局变量则返回true。

delete global_var; //false
delete global_novar; //true

若此时获取全局变量与隐式全局变量的值，则有所不同。全局变量仍然为1，隐式全局则是undefined。

typeof global_var; //'number'
typeof global_novar; //'undefined'

‘use sctrict’时，使用隐式变量直接报错。

for in循环:

for in称为“枚举”，用在对象的遍历上。数组用for循环。
使用for in时候可以使用hasOwnProperty过滤原型链上的属性(只会输出对象的属性值，会过滤掉原型链上的属性)。

var man = {
hands : 2,
legs : 2,
heads : 1
};

if (typeof Object.prototype.clone === "undefined") {
Object.prototype.clone = function () {};
};

for (var i in man) {
if (man.hasOwnPrototype(i)) {
console.log(i,":",man[i]);
};
};
将输出：
hands : 2
legs : 2
heads : 1

或者写成：

for (var i in man) {
if (Object.prototype.hasOwnPrototype.call(man,i)) {
console.log(i, ":", man[i]);
}
};

或者使用局部变量进行缓存：

var i, hasOwn = Object.prototype.hasOwnPrototype;
for (i in man) {
if (hasOwn.call(man, i)) {
console.log(i, ":", man[i]);
};
};

for与for in循环与if语句的连写：
//警告：通不过jslint的检测
var i, hasOwn = Object.prototype.hasOwnProperty;
for (var i in man) if (hasOwn.call(man, i)) {
console.log(i, ":", man[i])
}

避免使用eval()：

eval接受任意字符串。当事先知道代码时候可以使用:

var property = 'name';
alert(obj[property]);

代替：

var property = 'name';
alert(eval('obj.' + protety));

使用eval有安全隐患。特别是处理JSON时，可以使用内置的JSON.parse()，若不支持，则使用JSON.org

同样要避免给setInteral()、setTimeout()、Function()传递字符串。

//反面例子
setTimeout("myFunc()",1000);
setTimeout("myFunc(1,2)",1000);

//正面例子
setTimeout(myFunc,1000);
setTimeout(function () {
myFunc(1,2);()
},1000);

Function构造类似eval。如果必须使用eval可以使用new Function。因为，Function实在局部作用域中运行代码，避免影响全局：
console.log(typeof un); //undefined
console.log(typeof deux); //undefined
console.log(typeof trois); //undefined

var jsstring = 'var un = 1; console.log(un);'
eval(jsstring); //log:1

var jsstring = 'var deux = 2; console.log(deux);'
new Function(jsstring)(); //log:2

var jsstring = 'var trois = 3; console.log(trois);'
(function () {
eval(jsstring);
}()}; //log:3

console.log(typeof un); //number
console.log(typeof deux); //undefined
console.log(typeof trois); //undefined

eval可以干扰作用域链。Function()则不(Function()与new Function()是相同的)。

(function () {
var local = 1;
eval('local = 3; console.log(local)'); //log 3
console.log(local); //log 3
}());

(function () {
var local = 1;
Function('console.log(typeof local);')(); //log undefined
}());
